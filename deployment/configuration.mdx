Teletext Radio's configuration is primarily managed through JavaScript constants and JSON files. There are no environment variables or build-time configuration required.

## Core configuration files

### playlist.json

The `playlist.json` file defines the audio tracks for each program:

```json
{
  "tracks": [
    "/music/madrugada/track1.mp3",
    "/music/madrugada/track2.mp3",
    "/music/mañana/track1.mp3"
  ]
}
```

<Info>
The playlist is loaded dynamically by `radio-zara.js` at runtime. You can update it without redeploying the entire site.
</Info>

## Synchronization settings

The synchronization mechanism is configured in `radio-zara.js:138-160`:

```javascript
function calcularPosicionExacta() {
    const inicioTransmision = new Date('2025-01-01T03:00:00Z');
    const ahora = new Date();
    
    const segundosTranscurridos = Math.floor((ahora - inicioTransmision) / 1000);
    const segundosPorCancion = 180; // 3 minutes per track
    const segundosTotalPlaylist = currentPlaylist.length * segundosPorCancion;
    const posicionEnPlaylist = segundosTranscurridos % segundosTotalPlaylist;
    
    currentTrackIndex = Math.floor(posicionEnPlaylist / segundosPorCancion) % currentPlaylist.length;
    const segundoEnCancion = posicionEnPlaylist % segundosPorCancion;
    
    return {
        trackIndex: currentTrackIndex,
        segundoEnCancion: segundoEnCancion,
        track: currentPlaylist[currentTrackIndex]
    };
}
```

### Key synchronization parameters

<Note>
**`inicioTransmision`** - The reference timestamp for synchronization. All clients calculate their position relative to this moment.
</Note>

<Note>
**`segundosPorCancion`** - Duration of each track in seconds (default: 180 = 3 minutes). Adjust if your tracks have different lengths.
</Note>

### Adjusting sync for different track lengths

If your tracks vary in length, you'll need to modify the synchronization logic:

```javascript
// Example: Variable track lengths
const trackDurations = [180, 240, 195, 210]; // seconds

let totalSeconds = 0;
for (let i = 0; i < trackDurations.length; i++) {
    if (segundosTranscurridos < totalSeconds + trackDurations[i]) {
        currentTrackIndex = i;
        segundoEnCancion = segundosTranscurridos - totalSeconds;
        break;
    }
    totalSeconds += trackDurations[i];
}
```

<Warning>
Changing the synchronization reference timestamp (`inicioTransmision`) will cause all listeners to jump to different positions in the playlist.
</Warning>

## Timezone configuration

The radio operates on Argentina time (UTC-3). This is configured in `radio-zara.js:52-58`:

```javascript
function getArgentinaTime() {
    const now = new Date();
    const argentinaOffset = -3 * 60; // UTC-3 in minutes
    const localOffset = now.getTimezoneOffset();
    const offsetDiff = argentinaOffset + localOffset;
    return new Date(now.getTime() + offsetDiff * 60000);
}
```

To use a different timezone:

```javascript
// Example: UTC+2 (e.g., CEST)
const timezoneOffset = 2 * 60; // UTC+2 in minutes
```

## Mode switching: Playlist vs Live

Teletext Radio supports two modes defined in `index.html:199-204`:

```html
<!-- MODO NORMAL (playlist) - USÁ ESTE CUANDO NO TRANSMITÍS -->
<script src="radio-zara.js"></script>

<!-- MODO EN VIVO (BUTT) - USÁ ESTE CUANDO TRANSMITÍS -->
<!-- <script src="radio-zara-live.js"></script>-->
```

<Steps>
  <Step title="Playlist mode (default)">
    Uses `radio-zara.js` for synchronized playback of pre-recorded tracks
  </Step>
  
  <Step title="Live mode">
    Uses `radio-zara-live.js` to connect to a live stream (e.g., from BUTT - Broadcast Using This Tool)
  </Step>
</Steps>

Switch between modes by commenting/uncommenting the appropriate script tag.

## CORS and streaming headers

For audio streaming to work correctly, your server must send these headers:

```http
Access-Control-Allow-Origin: *
Access-Control-Expose-Headers: *
Cache-Control: no-cache, no-store, must-revalidate
```

These are automatically configured if using `stream.php`. For static hosting, configure your server:

### Apache (.htaccess)

```apache
<FilesMatch "\.(mp3|ogg|wav)$">
    Header set Access-Control-Allow-Origin "*"
    Header set Access-Control-Expose-Headers "*"
</FilesMatch>
```

### Nginx

```nginx
location ~* \.(mp3|ogg|wav)$ {
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Expose-Headers *;
}
```
