## getCurrentSchedule()

Determines which radio program should be playing based on the current Argentina time.

```javascript
function getCurrentSchedule()
```

### Return value

Returns a schedule object with the following properties:

- `name` (string): Internal program identifier
- `displayName` (string): User-facing program name
- `start` (string): Start time in 24-hour format (e.g., "06:00")
- `end` (string): End time in 24-hour format (e.g., "12:00")

### Usage example

```javascript
const schedule = getCurrentSchedule();
const displayName = schedule.displayName || programNames[schedule.name];
currentShow.textContent = displayName;
```

The function handles special cases:

- The "especial" program only runs on Fridays (day 5) and Saturdays (day 6)
- Programs that cross midnight (e.g., noche: 20:00 - 01:00) are handled correctly
- Falls back to the first schedule (madrugada) if no match is found

### Schedule data

The function uses the `scheduleData` configuration:

```javascript
const scheduleData = {
    "schedules": [
        {"name": "madrugada", "displayName": "Radio 404", "start": "01:00", "end": "06:00"},
        {"name": "mañana", "displayName": "Archivo txt", "start": "06:00", "end": "12:00"},
        {"name": "tarde", "displayName": "Telesoft", "start": "12:00", "end": "16:00"},
        {"name": "mediatarde", "displayName": "Floppy Disk", "start": "16:00", "end": "20:00"},
        {"name": "noche", "displayName": "Internet Archive", "start": "20:00", "end": "01:00"},
        {"name": "especial", "displayName": "Especiales txt", "start": "22:00", "end": "00:00"}
    ]
};
```

## getArgentinaTime()

Calculates the current time in Argentina timezone (UTC-3), adjusting from the user's local timezone.

```javascript
function getArgentinaTime()
```

### Return value

Returns a `Date` object representing the current Argentina time.

### Usage example

```javascript
const now = getArgentinaTime();
const currentTime = now.getHours() * 60 + now.getMinutes();
```

The function converts local time to Argentina time:

```javascript
const now = new Date();
const argentinaOffset = -3 * 60; // UTC-3 in minutes
const localOffset = now.getTimezoneOffset();
const offsetDiff = argentinaOffset + localOffset;
return new Date(now.getTime() + offsetDiff * 60000);
```

## formatTimeForDisplay()

Converts 24-hour time format to 12-hour format with AM/PM.

```javascript
function formatTimeForDisplay(timeStr)
```

<ParamField path="timeStr" type="string" required>
  Time string in 24-hour format (e.g., "14:30")
</ParamField>

### Return value

Returns a formatted string in 12-hour format with AM/PM (e.g., "2:30 PM").

### Usage example

```javascript
const formatted = formatTimeForDisplay("14:30"); // Returns "2:30 PM"
const range = `${formatTimeForDisplay(schedule.start)} - ${formatTimeForDisplay(schedule.end)}`;
```

The function handles edge cases:

- Midnight (00:00) displays as "12:00 AM"
- Noon (12:00) displays as "12:00 PM"
- Minutes are always padded to 2 digits

## updateDisplayInfo()

Updates the DOM elements that display the current program name and time range.

```javascript
function updateDisplayInfo()
```

### Return value

Does not return a value.

### Side effects

- Calls `getCurrentSchedule()` to get current program
- Updates `currentShow.textContent` with program name
- Updates `currentTimeName.textContent` with program name
- Updates `currentTimeRange.textContent` with formatted time range

### Usage example

```javascript
// Update immediately on page load
updateDisplayInfo();

// Update every minute
setInterval(updateDisplayInfo, 60000);
```

The function targets these DOM elements:

- `#currentShow`: Main display area
- `#currentTimeName`: Time section program name
- `#currentTimeRange`: Program time range

## generateScheduleCards()

Generates HTML cards for all programs in the schedule and renders them in the schedule grid.

```javascript
function generateScheduleCards()
```

### Return value

Does not return a value.

### Side effects

- Clears existing content in `.schedule-grid`
- Creates a `.schedule-card` div for each program in `scheduleData.schedules`
- Appends cards to the schedule grid container
- Uses `programDescriptions` for card description text

### Usage example

```javascript
// Called during initialization
generateScheduleCards();
```

Each generated card has this structure:

```html
<div class="schedule-card">
    <div class="schedule-time">6:00 AM - 12:00 PM</div>
    <div class="schedule-name">Archivo txt</div>
    <div class="schedule-desc">Programa matutino con energía y ritmos para comenzar el día.</div>
</div>
```

### Program descriptions

The function uses the `programDescriptions` configuration:

```javascript
const programDescriptions = {
    "madrugada": "Sonidos atmosféricos y experimentales para las primeras horas del día.",
    "mañana": "Programa matutino con energía y ritmos para comenzar el día.",
    "tarde": "Ritmos variados y selecciones especiales para acompañar la tarde.",
    "mediatarde": "Transición hacia la noche con sonidos más atmosféricos.",
    "noche": "Sesiones extendidas y atmósferas nocturnas para terminar el día.",
    "especial": "Programación especial viernes y sábados de 22:00 a 00:00."
};
```
