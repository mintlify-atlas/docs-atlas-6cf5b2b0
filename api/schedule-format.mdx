The schedule system in Teletext Radio defines time-based programming blocks throughout the day. It's configured in `radio-zara.js` and determines which show is currently playing.

## scheduleData object

The main schedule configuration object contains an array of schedule entries:

```javascript
const scheduleData = {
    "schedules": [
        {"name": "madrugada", "displayName": "Radio 404", "start": "01:00", "end": "06:00"},
        {"name": "mañana", "displayName": "Archivo txt", "start": "06:00", "end": "12:00"},
        {"name": "tarde", "displayName": "Telesoft", "start": "12:00", "end": "16:00"},
        {"name": "mediatarde", "displayName": "Floppy Disk", "start": "16:00", "end": "20:00"},
        {"name": "noche", "displayName": "Internet Archive", "start": "20:00", "end": "01:00"},
        {"name": "especial", "displayName": "Especiales txt", "start": "22:00", "end": "00:00"}
    ]
};
```

<ResponseField name="schedules" type="array" required>
  Array of schedule objects defining each programming block.
</ResponseField>

## Schedule object

Each schedule in the array has the following structure:

<ResponseField name="name" type="string" required>
  Internal identifier for the program. Used as a key to look up program names and descriptions.
  
  Valid values: `"madrugada"`, `"mañana"`, `"tarde"`, `"mediatarde"`, `"noche"`, `"especial"`
</ResponseField>

<ResponseField name="displayName" type="string" required>
  The public-facing name shown to users in the UI.
  
  Examples: `"Radio 404"`, `"Archivo txt"`, `"Telesoft"`
</ResponseField>

<ResponseField name="start" type="string" required>
  Start time in 24-hour format `"HH:MM"`.
  
  Example: `"06:00"`, `"22:00"`
</ResponseField>

<ResponseField name="end" type="string" required>
  End time in 24-hour format `"HH:MM"`. Can extend past midnight (e.g., `"01:00"` for a block starting the previous day).
  
  Example: `"12:00"`, `"01:00"`
</ResponseField>

## Program metadata objects

Two companion objects provide additional information about programs:

### programNames

Maps internal program names to display names:

```javascript
const programNames = {
    "madrugada": "Radio 404",
    "mañana": "Archivo txt", 
    "tarde": "Telesoft",
    "mediatarde": "Floppy Disk",
    "noche": "Internet Archive",
    "especial": "Especiales txt"
};
```

### programDescriptions

Provides descriptive text for each program:

```javascript
const programDescriptions = {
    "madrugada": "Sonidos atmosféricos y experimentales para las primeras horas del día.",
    "mañana": "Programa matutino con energía y ritmos para comenzar el día.",
    "tarde": "Ritmos variados y selecciones especiales para acompañar la tarde.",
    "mediatarde": "Transición hacia la noche con sonidos más atmosféricos.",
    "noche": "Sesiones extendidas y atmósferas nocturnas para terminar el día.",
    "especial": "Programación especial viernes y sábados de 22:00 a 00:00."
};
```

## Schedule resolution logic

The current schedule is determined by the `getCurrentSchedule()` function (`radio-zara.js:67-87`):

1. Gets current time in Argentina timezone (UTC-3)
2. Converts current time to minutes since midnight
3. Iterates through schedules to find matching time block
4. Special handling for "especial" program (only Fridays and Saturdays, day 5 and 6)
5. Handles schedules that cross midnight by adjusting end time
6. Returns matching schedule or defaults to first schedule ("madrugada")

## Time formatting

Schedule times are displayed to users in 12-hour format with AM/PM. The `formatTimeForDisplay()` function (`radio-zara.js:60-65`) converts:

- `"06:00"` → `"6:00 AM"`
- `"16:00"` → `"4:00 PM"`
- `"22:00"` → `"10:00 PM"`

## Notes

- All times are in Argentina timezone (UTC-3)
- The "especial" program only runs on Fridays and Saturdays
- Schedules can span midnight (e.g., "noche" runs 20:00-01:00)
- Schedule cards are generated dynamically in the UI from this data
- Current schedule updates every minute via `setInterval(updateDisplayInfo, 60000)`
