# Installation

This guide covers the complete installation and configuration of Teletext Radio, including all features like the schedule system, responsive design, and advanced player controls.

## Prerequisites

Before you begin, ensure you have:

<Steps>
  <Step title="Web server">
    A web server environment:
    - Apache
    - Nginx
    - Node.js with Express
    - Python HTTP server (for development)
  </Step>

  <Step title="Audio files">
    Your music library:
    - MP3 format (recommended)
    - Organized in directories
    - Proper file naming
  </Step>

  <Step title="Modern browser">
    For testing:
    - Chrome 90+
    - Firefox 88+
    - Safari 14+
    - Edge 90+
  </Step>
</Steps>

    ```javascript Time Functions
    // Get Argentina time (UTC-3)
    function getArgentinaTime() {
        const now = new Date();
        const argentinaOffset = -3 * 60;
        const localOffset = now.getTimezoneOffset();
        const offsetDiff = argentinaOffset + localOffset;
        return new Date(now.getTime() + offsetDiff * 60000);
    }
    
    // Get current schedule based on time
    function getCurrentSchedule() {
        const now = getArgentinaTime();
        const day = now.getDay();
        const currentTime = now.getHours() * 60 + now.getMinutes();
        
        for (const schedule of scheduleData.schedules) {
            // Skip special shows on non-weekend days
            if (schedule.name === "especial" && day !== 5 && day !== 6) continue;
            
            const start = schedule.start.split(':').map(Number);
            const end = schedule.end.split(':').map(Number);
            const startTime = start[0] * 60 + start[1];
            let endTime = end[0] * 60 + end[1];
            
            // Handle overnight shows
            if (endTime < startTime) endTime += 24 * 60;
            const adjustedCurrentTime = currentTime + (currentTime < startTime ? 24 * 60 : 0);
            
            if (adjustedCurrentTime >= startTime && adjustedCurrentTime < endTime) {
                return schedule;
            }
        }
        return scheduleData.schedules[0];
    }
    ```

## Configuration options

### Customize the transmission start time

The synchronization is based on a fixed start date. Change this in `radio-zara.js`:

```javascript
const inicioTransmision = new Date('2025-01-01T03:00:00Z');
```

<Note>
All listeners must use the same transmission start time for proper synchronization. Once set, don't change this value unless you want to reset the entire broadcast timeline.
</Note>

### Adjust track duration

By default, each track is assumed to be 180 seconds (3 minutes). Modify this if your tracks have different average lengths:

```javascript
const segundosPorCancion = 180; // Change to your average track length
```

### Customize the schedule

Edit the schedule configuration to match your programming:

```javascript
const scheduleData = {
    "schedules": [
        {
            "name": "morning",
            "displayName": "Morning Mix",
            "start": "06:00",
            "end": "12:00"
        },
        // Add more schedules
    ]
};
```

### Change timezone

The default timezone is Argentina (UTC-3). To use a different timezone:

```javascript
function getArgentinaTime() {
    const now = new Date();
    const argentinaOffset = -3 * 60; // Change to your UTC offset in minutes
    const localOffset = now.getTimezoneOffset();
    const offsetDiff = argentinaOffset + localOffset;
    return new Date(now.getTime() + offsetDiff * 60000);
}
```

## Deployment

### Production considerations

1. **Use a CDN** for audio files to reduce server load
2. **Enable HTTPS** for share functionality and mobile compatibility
3. **Compress audio files** to reduce bandwidth
4. **Set proper CORS headers** if serving audio from different domain
5. **Add caching headers** for static assets

### Example Nginx configuration

```nginx
server {
    listen 80;
    server_name radio.example.com;
    root /var/www/teletext-radio;
    index index.html;

    # Cache static assets
    location ~* \.(css|js|jpg|png|otf)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Don't cache HTML
    location / {
        add_header Cache-Control "no-cache";
    }

    # Stream audio with proper headers
    location /music/ {
        add_header Access-Control-Allow-Origin *;
        add_header Accept-Ranges bytes;
    }
}
```

## Troubleshooting

### Audio won't play

- Check browser console for errors
- Verify audio file paths in `playlist.json`
- Ensure files are accessible (check permissions)
- Test on desktop before mobile (mobile has stricter autoplay policies)

### Synchronization issues

- Verify all clients have the same transmission start time
- Check system clocks are accurate
- Clear browser cache and reload

### Schedule not updating

- Check timezone configuration
- Verify schedule times don't overlap
- Check browser console for schedule calculation logs
