Teletext Radio's programming, branding, and content can be customized by editing configuration objects in the JavaScript files and updating HTML content.

## Program names and schedules

All program configuration is managed in `radio-zara.js`. You can modify program names, descriptions, and time slots.

### Program names

Edit the `programNames` object in radio-zara.js:22-29:

```javascript
const programNames = {
    "madrugada": "Radio 404",
    "mañana": "Archivo txt", 
    "tarde": "Telesoft",
    "mediatarde": "Floppy Disk",
    "noche": "Internet Archive",
    "especial": "Especiales txt"
};
```

<Note>
The object keys ("madrugada", "mañana", etc.) are internal IDs that should match your music folder names. The values are the display names shown to users.
</Note>

### Schedule configuration

Define when each program airs by modifying the `scheduleData` object in radio-zara.js:40-49:

```javascript
const scheduleData = {
    "schedules": [
        {"name": "madrugada", "displayName": "Radio 404", "start": "01:00", "end": "06:00"},
        {"name": "mañana", "displayName": "Archivo txt", "start": "06:00", "end": "12:00"},
        {"name": "tarde", "displayName": "Telesoft", "start": "12:00", "end": "16:00"},
        {"name": "mediatarde", "displayName": "Floppy Disk", "start": "16:00", "end": "20:00"},
        {"name": "noche", "displayName": "Internet Archive", "start": "20:00", "end": "01:00"},
        {"name": "especial", "displayName": "Especiales txt", "start": "22:00", "end": "00:00"}
    ]
};
```

<Steps>
  <Step title="Set program ID">
    The `name` field should match a key in `programNames` and your music folder name.
  </Step>

  <Step title="Set display name">
    The `displayName` field appears in the schedule grid and player interface.
  </Step>

  <Step title="Define time range">
    Use 24-hour format for `start` and `end` times (e.g., "22:00" for 10 PM).
  </Step>

  <Step title="Handle overnight shows">
    For programs that cross midnight (like "noche": "20:00" to "01:00"), the code automatically handles the day transition.
  </Step>
</Steps>

## Adding a new program

<Steps>
  <Step title="Create music folder">
    Create a new folder under `music/` with your program name:

    ```bash
    mkdir music/mynewshow
    ```
  </Step>

  <Step title="Add to program names">
    Add an entry to `programNames`:

    ```javascript
    const programNames = {
        // ... existing programs
        "mynewshow": "My New Show"
    };
    ```
  </Step>

  <Step title="Add description">
    Define the program description:

    ```javascript
    const programDescriptions = {
        // ... existing descriptions
        "mynewshow": "An exciting new program with curated music."
    };
    ```
  </Step>

  <Step title="Add to schedule">
    Insert the schedule entry:

    ```javascript
    const scheduleData = {
        "schedules": [
            // ... existing schedules
            {"name": "mynewshow", "displayName": "My New Show", "start": "14:00", "end": "18:00"}
        ]
    };
    ```
  </Step>

  <Step title="Add music files">
    Copy MP3 or M4A files into `music/mynewshow/`.
  </Step>

  <Step title="Generate playlist">
    Run the playlist generator:

    ```bash
    python3 playlist_generator.py
    ```
  </Step>
</Steps>

## Special programs

The "especial" program only plays on specific days. This is controlled in radio-zara.js:73:

```javascript
if (schedule.name === "especial" && day !== 5 && day !== 6) continue;
```

This makes the special program only active on Fridays (5) and Saturdays (6).

<Warning>
Days are zero-indexed: Sunday = 0, Monday = 1, ..., Saturday = 6
</Warning>

To change which days the special program airs:

```javascript
// Only on Sundays:
if (schedule.name === "especial" && day !== 0) continue;

// Fridays, Saturdays, and Sundays:
if (schedule.name === "especial" && day !== 5 && day !== 6 && day !== 0) continue;
```

## Time zone configuration

The radio uses Argentina time (UTC-3). The conversion is handled in radio-zara.js:52-58:

```javascript
function getArgentinaTime() {
    const now = new Date();
    const argentinaOffset = -3 * 60;  // Argentina is UTC-3
    const localOffset = now.getTimezoneOffset();
    const offsetDiff = argentinaOffset + localOffset;
    return new Date(now.getTime() + offsetDiff * 60000);
}
```

To use a different time zone, change the offset:

```javascript
const argentinaOffset = -5 * 60;  // EST (UTC-5)
const argentinaOffset = 1 * 60;   // CET (UTC+1)
const argentinaOffset = 0;        // UTC
```
