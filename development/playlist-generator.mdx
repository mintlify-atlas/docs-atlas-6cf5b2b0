The `playlist_generator.py` script automates the creation of playlist.json files for each radio program. It scans music directories, extracts metadata, and generates synchronized playlists.

## Overview

The playlist generator reads program configuration, scans audio files in each program folder, calculates durations, and outputs JSON playlist files that the radio player uses for synchronized playback.

**Key features:**
- Automatic duration detection using mutagen library
- Fixed randomization seed for consistent playback across all listeners
- Support for MP3 and M4A audio formats
- Per-program playlist generation

## Requirements

The script requires Python 3 and the mutagen library for audio metadata extraction:

```bash
pip install mutagen
```

## Usage

### Basic usage

Run the script from your project root directory:

```bash
python3 playlist_generator.py
```

The script will:
1. Read the program configuration from `music/_programs.json`
2. Scan each program folder for audio files
3. Generate a `playlist.json` file in each program directory
4. Display progress and statistics

## How it works

### Configuration file

The script expects a configuration file at `music/_programs.json` with this structure:

```json
{
  "programs": [
    {
      "name": "Radio 404",
      "folder": "radio404",
      "schedule": "01:00-06:00"
    },
    {
      "name": "Archivo txt",
      "folder": "archivotxt",
      "schedule": "06:00-12:00"
    }
  ]
}
```

<Note>
If `_programs.json` doesn't exist, the script will display an error and exit. Create this file before running the generator.
</Note>

### Duration detection

The script uses two methods to determine track duration:

```python
def obtener_duracion_real(mp3_path):
    """Obtiene duración real en segundos"""
    try:
        audio = mutagen.File(mp3_path)
        if audio is not None:
            return int(audio.info.length)
    except:
        pass
    # Estimación por tamaño (1MB ≈ 1 minuto)
    size_mb = mp3_path.stat().st_size / (1024 * 1024)
    return int(size_mb * 60) if size_mb > 0 else 300
```

1. **Primary method:** Uses mutagen to read actual audio duration from file metadata
2. **Fallback method:** Estimates duration based on file size (1MB ≈ 1 minute)

### Randomization seed

The script uses a fixed seed for shuffling to ensure all listeners hear the same track order:

```python
SEMILLA_FIJA = 42  # Fixed random seed per program

# Orden aleatorio FIJO por programa
random.seed(SEMILLA_FIJA + hash(programa["folder"]) % 1000)
random.shuffle(tracks)
```

This creates deterministic "randomness" - the shuffle order is fixed but different for each program.

### Output format

The generated `playlist.json` file contains:

```json
{
  "program": "Radio 404",
  "folder": "radio404",
  "schedule": "01:00-06:00",
  "seed": 42,
  "total_duration": 8100,
  "total_tracks": 12,
  "tracks": [
    {
      "file": "track01.mp3",
      "duration": 245,
      "path": "music/radio404/track01.mp3"
    }
  ]
}
```

## Customization

### Change the randomization seed

Modify the seed constant to change the shuffle order:

```python
SEMILLA_FIJA = 42  # Change this number
```

### Add support for new audio formats

Extend the file extension list in playlist_generator.py:42:

```python
for ext in ['*.mp3', '*.MP3', '*.m4a', '*.M4A', '*.ogg', '*.OGG']:
    archivos_mp3.extend(carpeta.glob(ext))
```

### Adjust duration estimation

Change the fallback calculation in playlist_generator.py:29:

```python
# Current: 1MB = 1 minute
return int(size_mb * 60)

# For higher bitrate files: 1MB = 45 seconds
return int(size_mb * 45)
```

## Troubleshooting

<Warning>
If the script fails with "No module named 'mutagen'", install the dependency:

```bash
pip install mutagen
```
</Warning>

### Common issues

**No files found in program folder**

```
❌ No hay archivos de audio en 'radio404'
```

Solution: Verify the folder exists under `music/` and contains MP3 or M4A files.

**Configuration file missing**

```
❌ ERROR: No existe music/_programs.json
```

Solution: Create the configuration file with your program definitions.

**Permission errors**

Solution: Ensure the script has write permissions to the music subdirectories:

```bash
chmod -R u+w music/
```
